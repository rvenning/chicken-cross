# Chicken Cross 🐔

A Crossy-Road-style family game. Hop your bird across roads, rivers and railways —
dodge the cars, ride the logs, and don't argue with the trains.

**Play it:** https://rvenning.github.io/chicken-cross/

## Features
- 🗺️ 10 worlds × 4 levels (Sunny Meadows to the lava rivers of Volcano Finale), plus ♾️ Endless mode
- 🚂 Railway crossings with warning lights, lowering gates, and very fast trains
- 🪙 Coins to collect; 💛 spend 50 to revive where you died
- ⭐ Up to 3 stars per level (finish + coin bonuses)
- 🐔🦆🐧 Your profile avatar is who you play — every bird has its own hand-drawn sprite (mallard, penguin, owl, flamingo, peacock…)
- 👨‍👩‍👧 Pick-a-name family profiles — no passwords; optional 4-digit PIN per profile with an admin override
- 🏆 Shared family leaderboard (best distance)
- 🔊 All sound effects synthesized with WebAudio (no asset downloads)
- ☁️ Firebase Firestore sync (falls back to localStorage offline)
- 📱 iPad-first controls: tap to hop, swipe to steer (fires mid-gesture, Crossy-Road style); works with keyboard/WASD on desktop
- 📲 Installable PWA: "Add to Home Screen" button, offline play via service worker
- 🌊 Watery water, as requested by Nathan

## Built on gamekit
Profiles, PINs, storage/sync, the sound engine, base UI styles and the PWA kit come from
[rvenning/gamekit](https://github.com/rvenning/gamekit), vendored into `lib/`.
To pull in a newer kit version:

```powershell
# from the gamekit repo
node tools/sync-to-game.js "D:\OneDrive\Documents\Claude Code\chicken-cross"
# then test, bump the CACHE version in sw.js, and commit
```

Everything game-specific — the canvas engine, worlds/levels, bird sprites, jingles —
lives in `index.html`.

## PWA files
- `manifest.json` — app identity: name, icons, standalone display, theme colours
- `sw.js` — service worker; precaches the app shell (including `lib/`), then network-first with cache fallback
- `icons/` — 192/512 PNG app icons plus a maskable variant, generated by `tools/make-icons.js` (no image tools needed — pure Node)

## Local development
No build step — plain HTML/JS. Serve the folder with any static server:

```powershell
npx http-server . -p 8095 -c-1   # http://localhost:8095
```

Regenerate the icons after changing the sprite art:

```powershell
node tools/make-icons.js
```

Deploys are just pushes to `main` (GitHub Pages).

## Storage
Progress lives in `localStorage` (keys `cc_*`) and mirrors to Cloud Firestore
(collection `chickencross`, same Firebase project as WordVoyage). The Firebase API
key in `index.html` is public by design — it only identifies the project; access is
limited by Firestore security rules, and the key is referrer-restricted to the
game's domains and locked to the Firestore API.
